Q : Conventional Commits란 ? 
A : 커밋 메시지를 일정한 형식으로 작성하는 규칙
자주 쓰는 타입:

┌──────────┬───────────────────────────────────────────┐
│   타입   │                 언제 쓰나                 │
├──────────┼───────────────────────────────────────────┤
│ feat     │ 새 기능 추가                              │
├──────────┼───────────────────────────────────────────┤
│ fix      │ 버그 수정                                 │
├──────────┼───────────────────────────────────────────┤
│ docs     │ 문서 수정                                 │
├──────────┼───────────────────────────────────────────┤
│ style    │ 코드 포매팅, 세미콜론 등 (기능 변화 없음) │
├──────────┼───────────────────────────────────────────┤
│ refactor │ 리팩토링                                  │
├──────────┼───────────────────────────────────────────┤
│ test     │ 테스트 추가/수정                          │
├──────────┼───────────────────────────────────────────┤
│ chore    │ 빌드, 설정 파일 수정 등                   │
└──────────┴───────────────────────────────────────────┘
