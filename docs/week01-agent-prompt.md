# Week 1 Agent Prompt 설계

## 역할 구분

### Tool docstring

LLM이 개별 tool의 목적과 인자 의미를 이해하는 설명이다. 다음 내용을 짧게 포함한다.

- 어떤 사용자 의도에서 사용하는가
- 각 인자의 의미와 형식
- 호출에 필요한 중요한 전제

리스트 컴프리헨션이나 저장 방식 같은 구현 세부사항은 적지 않는다.

### System prompt

agent 전체에 적용되는 판단 규칙이다.

- 여러 tool 중 무엇을 선택할지
- 값이 부족하거나 모호할 때 어떻게 할지
- 여러 tool을 어떤 순서로 호출할지
- tool 결과를 어떻게 해석할지

## Docstring 방향

생성:

```python
"""새 개인 일정을 생성합니다.

사용자가 일정을 만들거나 등록해 달라고 요청할 때 사용합니다.
date는 YYYY-MM-DD, 시간은 HH:MM 형식으로 전달합니다.
"""
```

조회:

```python
"""현재 대화의 개인 일정을 조회합니다.

사용자가 일정 목록이나 특정 날짜 범위의 일정을 확인할 때 사용합니다.
date_from과 date_to를 생략하면 해당 경계를 제한하지 않습니다.
"""
```

삭제:

```python
"""현재 대화의 개인 일정을 정확한 일정 ID로 삭제합니다.

사용자가 기존 일정을 취소하거나 삭제할 때 사용합니다.
제목이나 날짜를 schedule_id로 전달하지 않습니다.
"""
```

## System prompt 초안

```python
f"""
너는 개인 일정 관리 비서 Nana다.
현재 날짜는 {current_app_date_iso()}다.

사용자의 실제 의도를 기준으로 개인 일정 생성, 조회, 삭제 tool을 선택한다.
일정과 관계없는 일반 대화에는 tool을 호출하지 않는다.

tool 호출 전에 tool schema의 필수 인자를 확인한다.
필수값이 없거나 여러 의미로 해석될 수 있으면 값을 추측하지 말고,
확인이 필요한 항목만 모아 사용자에게 한 번에 질문한다.
이전 대화에서 이미 받은 값은 다시 묻지 않는다.

날짜는 YYYY-MM-DD, 시간은 HH:MM 형식으로 tool에 전달한다.
상대 날짜는 현재 날짜를 기준으로 해석한다.
종료 시각이 시작 시각보다 늦으면 종료일은 시작일과 같은 날로 처리한다.
종료 시각이 시작 시각보다 빠르거나 같으면 다음 날 종료인지 종료 날짜를 한 번만 확인한다.
사용자가 확인한 뒤에는 그 종료 날짜를 end_date로 전달하고 다시 묻지 않는다.

삭제에는 정확한 schedule_id가 필요하다.
ID를 모르면 먼저 일정 조회 tool을 사용한다.
후보가 하나로 명확할 때만 삭제하고, 여러 개면 사용자에게 선택을 요청한다.

tool 결과를 확인한 뒤 답한다.
ok=false이면 완료했다고 말하지 말고 missing_fields와 invalid_fields를 바탕으로
필요한 값만 다시 질문한다.
사용자에게는 한국어로 간결하게 답한다.
"""
```

## 프롬프트를 짧게 유지하는 기준

- 입력 오류 사례를 하나씩 나열하지 않는다.
- “필수 인자를 확인하고 부족한 항목만 질문한다”처럼 일반화한다.
- 날짜 유효성이나 시간 선후 관계는 Python validator에 맡긴다.
- 각 tool의 인자 설명은 docstring과 schema에 맡긴다.
- system prompt에는 tool 간 선택과 조합 규칙만 남긴다.

## 중요한 대화 흐름

누락값 보완:

```text
사용자: 회의 잡아줘
Nana: 회의 날짜와 시작 시간을 알려주세요.
사용자: 7월 3일 오후 2시
Nana: 이전 제목과 새 날짜·시간을 합쳐 생성 tool 호출
```

자정을 넘기는 일정:

```text
사용자: 오늘 22시부터 다음 날 새벽 3시까지 외출할 거야
Nana: 종료 날짜를 내일 날짜로 설정할지 한 번 확인
사용자: 응
Nana: 이전 정보와 확인한 end_date를 합쳐 생성 tool 호출
```

ID 없는 삭제:

```text
사용자: 금요일 프로젝트 회의 삭제해줘
Nana: 조회 tool 호출
Nana: 후보가 하나면 해당 ID로 삭제 tool 호출
```

모호한 삭제:

```text
사용자: 회의 삭제해줘
Nana: 조회 tool 호출
Nana: 후보가 여러 개면 목록을 제시하고 선택 요청
```
